CAN Bridge Config Web Manual

Guide for the M5 AtomS3R A/B CAN wireless bridge web tool.

Open the app

1. What this tool does

  • Load and save config JSON files
  • Import CAN IDs from trace files
  • Import live CAN IDs from unit A over USB
  • Edit `Allow` and `High` lists
  • Build strict configs from observed traffic
  • Write settings to unit A over USB
  • Sync settings from A to B

2. Main workflow

  1. Open the web app.
  2. Choose an input source:
    • Load JSON for an existing config
    • Import Trace for a saved trace file
    • USB Connect then USB Import Live for live traffic from unit A
  3. Build the config:
    • Trace -> Strict for exact IDs from the latest imported data
    • Auto By Count for frame-count-based ranking
    • Auto By Rate for update-rate-based ranking
  4. Review `Allow` and `High`.
  5. Use USB Write to write to unit A.
  6. Let unit A sync the setting to unit B.

3. Recommended operating method

  1. Run the bridge in FILTER=ALL.
  2. Capture real traffic from the actual CAN line.
  3. Import the data into the web app.
  4. Use Trace -> Strict to build the base `Allow` list.
  5. Adjust `High` only after the exact traffic set is correct.

This is the safest method when reliability matters more than speed.

4. Live import

  1. Connect unit A by USB.
  2. Click USB Connect.
  3. Click USB Clear Live.
  4. Let real CAN traffic flow for a few seconds.
  5. Click USB Import Live.

If the app shows No live IDs captured, unit A did not observe any CAN frames during that capture window.

For reliable live import, connect unit A directly to the same CAN point you want to observe.

5. Save and write behavior

  • Save JSON saves the current `Allow` and `High` lists.
  • The left `Candidates` list is not the saved output.
  • After importing trace or live data, build the actual config before saving.
  • USB Write sends the current config to unit A.

6. ECU Master ADU note

Runtime CAN bridging and ADU Client PC communication are not the same use case.

  • For setup, firmware update, or ADU Client communication, use direct wired connection.
  • Use the wireless bridge mainly for runtime display and selected CAN transfer.

7. Current limits

  • The M5 config limit is 128 IDs per list.
  • Wireless security settings are not finished yet.
  • Future items:
    • channel
    • network ID
    • shared key
    • safe staged A/B wireless switch
